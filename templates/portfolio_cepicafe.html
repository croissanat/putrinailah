Project Overview

Developed a comprehensive restaurant management system for Cepi Cafe featuring online ordering, menu management, order tracking, and customer experience recording. The system includes both customer-facing interfaces and administrative dashboards.

Designed and implemented a robust relational database with 8 main entities following 3NF normalization standards, with complex relationships including one-to-one, one-to-many, and many-to-many.

🗃️ Database Schema

Main Entities

  • • Menu: Food/beverage items
  • • Customer: User information
  • • Order: Transaction records
  • • Order Status: Lifecycle states
  • • Customer Data: Additional info
  • • Experience: Ratings/feedback
  • • Order History: Past orders
  • • Bank: Payment information

Key Relationships

  • • Order to Customer: Many-to-One
  • • Order to Menu: One-to-Many
  • • Experience to Order: One-to-One
  • • Order Status to Order: One-to-One
  • • Customer Data to Customer: One-to-Many
  • • Order History to Customer: Many-to-One
  • • Order History to Order: One-to-One

Development Process

1️⃣ Requirements Analysis

Conducted analysis of cafe operations, identified key features, and defined business rules for the database system.

2️⃣ Database Design

Created ERD with 8 entities and multiple relationships. Implemented normalization to 3NF with proper keys.

3️⃣ Backend Development

Implemented PHP/Laravel backend with MySQL database for menu, orders, and customer experience.

4️⃣ Frontend Implementation

Developed responsive interfaces using HTML, CSS, JavaScript, and Bootstrap with AJAX for data loading.
